        Introduction # There have been several approaches to dynamic reconfiguration of consensus algorithms. Dynamic Paxos (Multi-Paxos that supports reconfig) limits the number of concurrent proposals as a pipeline length parameter $\alpha > 0$, therefore a configuration change chosen at instance $i$ does not take effect until instance $i + \alpha$.
